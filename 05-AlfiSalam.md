## 🔹 Apa itu Pola PREP (dengan Foreground)?

**PREP = Point → Reason → Example → Point (ulang/penegasan)**
Dengan **foreground**, berarti kamu langsung kasih **inti jawaban di awal (Point)**, baru dijelaskan.

---

## 🔹 Struktur Lengkapnya

1. **Point (Foreground / Inti)**

   * Jawaban langsung, singkat, tegas.
2. **Reason (Alasan)**

   * Kenapa pernyataan itu benar/logis.
3. **Example (Contoh)**

   * Ilustrasi nyata biar mudah dipahami.
4. **Point (Penegasan ulang)**

   * Kesimpulan yang memperkuat jawaban.

---

## 🔹 Contoh (Topik: Fungsi GitHub)

### 1. Point (Inti di depan)

GitHub berfungsi sebagai platform untuk menyimpan, mengelola, dan berkolaborasi dalam pengembangan kode program.

### 2. Reason (Alasan)

Karena GitHub menggunakan sistem version control (Git) yang memungkinkan banyak orang bekerja pada proyek yang sama tanpa konflik, serta menyimpan riwayat perubahan kode.

### 3. Example (Contoh)

Misalnya, dalam sebuah tim developer, satu orang mengerjakan fitur login, sementara yang lain mengerjakan tampilan. Semua perubahan bisa digabungkan di GitHub tanpa saling menimpa.

### 4. Point (Penegasan ulang)

Jadi, GitHub adalah alat penting untuk kolaborasi dan manajemen kode dalam pengembangan software.

## 🔹 Contoh Kedua (Topik: Kewirausahaan)

### Point

Kewirausahaan adalah kemampuan menciptakan usaha dengan memanfaatkan peluang dan berani mengambil risiko.

### Reason

Hal ini penting karena seorang wirausaha harus mampu melihat peluang pasar dan mengambil keputusan meskipun ada ketidakpastian.

### Example

Contohnya, seseorang membuka usaha ayam krispi seperti brand “Celap Celup” karena melihat banyak peminat makanan cepat saji di sekitarnya.

### Point

Dengan demikian, kewirausahaan menuntut kreativitas, keberanian, dan kemampuan membaca peluang.

---

## 🔹 Tips Supaya PREP Kamu Kuat

* Point harus **langsung jawab pertanyaan**
* Reason jangan bertele-tele
* Example harus **nyata dan relevan**
* Point terakhir = **ringkasan kuat, bukan copy-paste**
