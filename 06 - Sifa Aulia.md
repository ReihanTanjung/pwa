Point (Inti Pembahasan)

Gambar ini membahas perbandingan jenis pengembangan aplikasi mobile, yaitu:

1. Native App
2. Hybrid App
3. PWA (Progressive Web App)

Fokusnya adalah membandingkan dari sisi akses hardware, performa, biaya, dan teknologi yang digunakan.

Reason (Penjelasan Detail)

 1. Native App

Native adalah aplikasi yang dibuat khusus untuk sistem operasi tertentu (Android/iOS).

Ciri yang terlihat di papan:

 Bahasa: Kotlin, Java (Android)
 Akses hardware: penuh (HTC = Hardware seperti Bluetooth, Camera, dll)
 Performa: paling tinggi
 Biaya: besar (harus buat versi Android & iOS terpisah)

Tambahan catatan:

 Mendukung teknologi seperti AR/VR dan game dengan baik
 Cocok untuk aplikasi kompleks

 2. Hybrid App

Hybrid adalah gabungan antara web dan native.

Ciri yang tertulis:

 Menggunakan plugin untuk akses hardware
 Dibuat dengan HTML, CSS, JavaScript
 Performa: sedang (tidak secepat native)
 Biaya: lebih murah dibanding native

Penjelasan:

 Aplikasi dibuat sekali, bisa dijalankan di banyak platform
 Menggunakan “pembungkus” (wrapper) agar bisa jalan seperti aplikasi native

 3. PWA (Progressive Web App)

PWA adalah website yang dibuat menyerupai aplikasi mobile.

Ciri yang tertulis:

 Teknologi: HTML, CSS, JavaScript
 Tidak perlu install dari Play Store
 Akses hardware: terbatas
 Performa: paling ringan, tapi fitur terbatas

Tambahan:

 Bisa diakses lewat browser
 Menggunakan API untuk beberapa fitur (kamera, notifikasi, dll)

 4. Perbandingan Utama (yang terlihat di kiri papan)

Beberapa aspek yang dibandingkan:

 Akses hardware
 Performa
 Biaya
 Desain / fleksibilitas

Kesimpulan dari tabel:

 Native → terbaik performa & fitur, tapi mahal
 Hybrid → tengah-tengah
 PWA → paling ringan & murah, tapi terbatas

 Example (Contoh Kasus)

Misalnya ingin membuat aplikasi:

1. Game atau aplikasi AR/VR

    Pilihan: Native
    Karena butuh performa tinggi dan akses hardware penuh

2. Aplikasi bisnis sederhana (misalnya katalog produk)

    Pilihan: Hybrid
    Lebih hemat biaya dan cukup untuk kebutuhan standar

3. Website toko online yang ingin seperti aplikasi

    Pilihan: PWA
    Bisa dibuka tanpa install, ringan, dan cepat


 Point (Kesimpulan)

Gambar ini menekankan bahwa pemilihan jenis aplikasi harus disesuaikan dengan kebutuhan:

 Native untuk performa tinggi dan fitur lengkap
 Hybrid untuk efisiensi pengembangan lintas platform
 PWA untuk solusi ringan berbasis web

Semakin tinggi kebutuhan performa dan fitur, maka semakin kompleks dan mahal teknologi yang digunakan.
